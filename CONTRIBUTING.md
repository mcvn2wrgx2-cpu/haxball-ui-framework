# Contributing

Thank you for your interest in contributing to HaxBall UI Framework.

## Getting Started

1. Fork the repository.
2. Create a new branch for your changes.
3. Make your modifications.
4. Test your changes locally.
5. Open a Pull Request.

## Guidelines

* Keep the framework lightweight and modular.
* Follow the existing code style.
* Avoid unnecessary dependencies.
* Write clear and descriptive commit messages.
* Focus on maintainability and performance.

## Reporting Issues

If you find a bug or have a feature request, please open an issue and provide as much detail as possible.

## Pull Requests

Before submitting a Pull Request:

* Ensure your code works correctly.
* Avoid unrelated changes in the same PR.
* Keep changes focused and easy to review.

## Questions

Feel free to open a discussion or issue if you need help or want to propose an idea.

Thank you for helping improve HaxBall UI Framework.
